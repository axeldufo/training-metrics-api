# TDD-007 ‚Äî LoadReport

## Context
Weekly training load aggregate per athlete per week. Persisted in database ‚Äî 
source of truth for weekly Foster load. Fed by TrainingSession domain events.
Consumed by WeeklyReport (TDD-008) for ACWR calculation and load/wellness correlation.
Current week included ‚Äî coach can consult in-progress week load at any time.

### Domain Object
`LoadReport` ‚Äî Java record in domain/. Never persisted directly ‚Äî constructed
either from DB via PersistenceMapper or on-the-fly in service.

```java
public record LoadReport(
    long athleteId,
    LocalDate weekStartDate,
    int totalFosterLoad,
    int sessionCount,
    LocalDateTime updatedAt
) {}
```

**Domain invariants (enforced in compact constructor ‚Äî DomainValidationException ‚Üí 400) :**
- `weekStartDate` not null + must be a Monday (no silent normalization)
- `totalFosterLoad` >= 0
- `sessionCount` >= 0
- `sessionCount == 0` iff `totalFosterLoad == 0` ‚Äî a report with sessions and zero load,
  or zero sessions and non-zero load, is a domain error
- `updatedAt` nullable ‚Äî null signals an on-the-fly report (no sessions persisted yet)

### JPA Entity
`LoadReportJpaEntity` in repository/ ‚Äî standard @Entity with @UpdateTimestamp on updatedAt.


### New exception (domain/exception/)
- `LoadReportNotFoundException(long athleteId, LocalDate weekStartDate)` ‚Üí 404
- Used exclusively by GET latest ‚Äî the only endpoint where absence of any report
  is a true missing resource (athlete has no load data at all)

## Domain Events
No new events ‚Äî reuses `TrainingSessionCreatedEvent`, `TrainingSessionUpdatedEvent`,
`TrainingSessionDeletedEvent` already published by `TrainingSessionServiceImpl`.
Load report recalculation merged into TrainingSessionEventHandler ‚Äî no separate handler.

Event flow:
1. Compute `weekStartDate` = Monday of `event.date()`
2. Fetch all sessions for that athlete on that week via
   `TrainingSessionRepository.findByAthleteIdAndPeriod(athleteId, weekStartDate, weekStartDate.plusDays(6))`
3. If sessions exist ‚Üí calculate `totalFosterLoad` + `sessionCount` ‚Üí `LoadReportRepository.save()`
4. If no sessions remain ‚Üí `LoadReportRepository.deleteByAthleteIdAndWeekStartDate()`

A zero-load report is never persisted ‚Äî it is always computed on-the-fly on GET.

## API Contract

### Endpoints
| Method | URL | Query params | Response | Success | Errors |
|---|---|---|---|---|---|
| GET | /v1/athletes/{id}/reports/load | `weekStartDate` (required) | `LoadReportResponse` | 200 | 400, 401 |
| GET | /v1/athletes/{id}/reports/load/latest | ‚Äî | `LoadReportResponse` | 200 | 401, 404 |
| GET | /v1/athletes/{id}/reports/load | `from` (required), `to` (optional) | `List<LoadReportResponse>` | 200 | 400, 401 |

Query params `weekStartDate` and `from`/`to` : `@PastOrPresent` ‚Äî future weeks rejected at HTTP boundary.

The two `GET /v1/athletes/{id}/reports/load` endpoints are implemented as two separate
controller methods discriminated by Spring MVC params:
- `@GetMapping(params = "weekStartDate")` ‚Üí single LoadReportResponse
- `@GetMapping(params = "from")` ‚Üí List<LoadReportResponse>

Note: GET by weekStartDate never returns 404 ‚Äî a valid week with no sessions returns a
zero-value report (totalFosterLoad=0, sessionCount=0, updatedAt=null). 404 is reserved
for GET latest when no report exists at all.

### DTO

- `int sessionCount`
- `LocalDateTime updatedAt` ‚Äî null if on-the-fly (no sessions for this week)

## Service flow

### GET by weekStartDate
GET /v1/athletes/{id}/reports/load?weekStartDate=2026-05-19
‚Üí LoadReportController ‚Äî @PastOrPresent on weekStartDate
‚Üí athleteService.findById(athleteId, coachId) ‚Äî Coach‚ÜíAthlete ownership
‚Üí LoadReportService.findByAthleteIdAndWeekStartDate(athleteId, weekStartDate)
‚Üí LoadReportRepository.findByAthleteIdAndWeekStartDate()
‚Üí found ‚Üí return persisted report
‚Üí not found ‚Üí TrainingSessionRepository.findByAthleteIdAndPeriod(
athleteId, weekStartDate, weekStartDate.plusDays(6))
‚Üí sessions found ‚Üí loadReportCalculator.calculate(athleteId, weekStartDate, sessions)
                   ‚Üí return LoadReport (not persisted)
‚Üí no sessions ‚Üí return LoadReport(totalFosterLoad=0, sessionCount=0, updatedAt=null)
‚Üí LoadReportWebMapper.domainToResponse(report)
‚Üí ResponseEntity.ok(response)

### GET latest
GET /v1/athletes/{id}/reports/load/latest
‚Üí LoadReportController
‚Üí athleteService.findById(athleteId, coachId) ‚Äî Coach‚ÜíAthlete ownership
‚Üí LoadReportService.findLatestByAthleteId(athleteId)
‚Üí LoadReportRepository.findLatestByAthleteId()
‚Üí found ‚Üí return
‚Üí empty ‚Üí throw LoadReportNotFoundException ‚Üí 404
‚Üí LoadReportWebMapper.domainToResponse(report)
‚Üí ResponseEntity.ok(response)

### GET by period
GET /v1/athletes/{id}/reports/load?from=2026-04-01&to=2026-05-26
‚Üí LoadReportController ‚Äî @PastOrPresent on from and to, validate from <= to ‚Üí 400
‚Üí athleteService.findById(athleteId, coachId) ‚Äî Coach‚ÜíAthlete ownership
‚Üí LoadReportService.findByAthleteIdAndPeriod(athleteId, from, to)
‚Üí LoadReportRepository.findByAthleteIdAndWeekStartDateBetween()
‚Üí returns list (empty list if no reports in range ‚Äî no 404)
‚Üí LoadReportWebMapper.domainToResponse() on each element
‚Üí ResponseEntity.ok(list)


### Event flow
TrainingSession created/updated/deleted
‚Üí TrainingSessionEventHandler
‚Üí weekStartDate = Monday of event.date()
‚Üí sessions = TrainingSessionRepository.findByAthleteIdAndPeriod(
athleteId, weekStartDate, weekStartDate.plusDays(6))
‚Üí if sessions empty ‚Üí LoadReportRepository.deleteByAthleteIdAndWeekStartDate()
‚Üí else ‚Üí LoadReportRepository.save(
     loadReportCalculator.calculate(athleteId, weekStartDate, sessions))

## Decisions
- LoadReport persisted ‚Äî aggregated weekly data, avoids recalculating from all sessions on every request
- `updated_at` only (no `created_at`) ‚Äî operational tracing of last recalculation; managed by
  @UpdateTimestamp. Distinguishes persisted reports (updatedAt set) from on-the-fly reports
  (updatedAt=null) in the API response
- Zero-load report never persisted ‚Äî GET has no write side effect (idempotent).
  On-the-fly calculation is a defensive fallback; in normal operation the event chain
  ensures the report is always in DB when sessions exist
- GET by weekStartDate never returns 404 ‚Äî absence of sessions is information, not an error
- GET latest returns 404 if no report exists ‚Äî true missing resource (athlete has no data at all)
- Future weekStartDate ‚Üí 400 at HTTP boundary, not 404 ‚Äî invalid input, not missing resource
- No Redis cache ‚Äî weekly data changes only on session events; DB read sufficient for this access pattern
- `LoadReportEventHandler` in service/ ‚Äî consistent with `TrainingSessionEventHandler` placement
- weekStartDate must be a Monday ‚Äî domain invariant, DomainValidationException ‚Üí 400, no silent normalization
- sessionCount == 0 iff totalFosterLoad == 0 ‚Äî domain invariant, incoherent report is a domain error
- Current week included ‚Äî coach can consult in-progress week (e.g. Saturday afternoon preview)
- LoadReport does NOT calculate ACWR ‚Äî raw weekly aggregate only; ACWR computed in WeeklyReport (TDD-008)
- Load calculation delegated to LoadReportCalculator (domain/) ‚Äî
  business logic (Foster load sum = week's load) belongs in domain, not in event handler
