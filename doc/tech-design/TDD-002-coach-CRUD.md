# TDD-002 ‚Äî Coach

## Context
Second domain. Introduces the Coach entity and its relationship with Athlete.
Establishes the multi-coach model where one Coach owns multiple Athletes.
Coach creation is handled via authentication (see TDD-005).

## Data Model

### Flyway Migrations
- `V2__create_coach_table.sql` ‚Äî table coach
- `V3__add_coach_id_to_athlete.sql` ‚Äî adds coach_id FK to athlete table
- `V5__add_email_and_password_to_coach.sql` ‚Äî adds email and hashed_password columns

### Table: coach
| Column | Type | Constraints |
|---|---|---|
| id | BIGINT GENERATED BY DEFAULT AS IDENTITY | PK |
| name | VARCHAR(100) | NOT NULL |
| email | VARCHAR(255) | NOT NULL UNIQUE |
| hashed_password | VARCHAR(255) | NOT NULL |

### Table: athlete (updated)
| Column | Type | Constraints |
|---|---|---|
| coach_id | BIGINT | nullable, FK ‚Üí coach(id) |

### Domain Entity
`Coach` ‚Äî pure POJO, Lombok `@Getter @EqualsAndHashCode(exclude="id") @ToString`.
Final fields except `id`. Manual constructor with domain invariants on `name` and `email`.

## Domain Interfaces

## Dependencies
No new dependencies.

## Impact on existing code
- `Athlete` domain ‚Äî `coachId` field added (nullable `Long`)
- `AthleteJpaEntity` ‚Äî `@ManyToOne(fetch = FetchType.LAZY)` relation to `CoachJpaEntity`
- `AthletePersistenceMapper.domainToEntity()` ‚Äî phantom `CoachJpaEntity` via builder for FK

## Decisions
- `CoachJpaEntity` uses `@Builder` + `@NoArgsConstructor` + `@AllArgsConstructor`
- Phantom entity pattern: `CoachJpaEntity.builder().id(coachId).build()` ‚Äî Hibernate only needs the id to persist the FK
- Domain invariants applied to `Coach`: null/blank check on `name` in constructor ‚Äî defensive, for future non-HTTP entry points (Kafka, gRPC)
- `updateName` uses `@Modifying(clearAutomatically = true) @Query` JPQL ‚Äî partial update, avoids overwriting email/hashed_password
- Coach creation handled exclusively via `POST /v1/auth/register` (see TDD-005)
- `email` in `Coach` domain ‚Äî needed for future async notifications (Kafka alerts)
- `hashed_password` only in `CoachJpaEntity` ‚Äî never in domain, never in response
- `GET /coaches/me`, `PUT /coaches/me`, `DELETE /coaches/me` ‚Äî id comes from JWT token, not path variable; ownership is implicitly guaranteed by the token
- `findAll()` removed ‚Äî a coach only manages their own profile
- `CoachService.findAll()` removed accordingly

## Out of scope
- Coach authentication and JWT (TDD-005)
- Profile update (email/password change) ‚Äî future ProfileController
- Pagination
