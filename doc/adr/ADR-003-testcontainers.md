# ADR-003 — Testcontainers for Integration Tests

## Status
Accepted

## Context
Phase 1 used Mockito exclusively for repository layer tests — adapters mocked
JpaRepository and PersistenceMapper. This approach verifies delegation logic but
cannot catch:
- SQL errors generated by Hibernate against real PostgreSQL
- Flyway migration correctness
- FK constraint violations
- Custom Spring Data derived queries (e.g. `findAllByAthleteId`)
- Routing errors only visible with a real HTTP stack

H2 in-memory was used for `contextLoads()` — rejected as a general test database
because H2 is not PostgreSQL: different SQL dialect, different type handling,
different constraint enforcement.

## Decision
Use Testcontainers with a real PostgreSQL 16 container for repository integration
tests (`*IT.java`). Remove H2 entirely.

**Shared container strategy:** `PostgresTestContainersConfiguration` — a single
`@TestConfiguration` bean with `@ServiceConnection` that starts one PostgreSQL
container per JVM, shared across all IT test classes.

**Test isolation:** `@Transactional` at class level — automatic rollback after each
test, no manual cleanup needed. Exception: tests verifying FK constraint violations
use `@Transactional(propagation = Propagation.NOT_SUPPORTED)` with manual cleanup,
because PostgreSQL only checks FK constraints at commit time.

**Naming convention:** `*IT.java` — Maven Failsafe convention, separates slow
integration tests from fast unit tests. Currently both run with `./mvnw test` since
Failsafe is not explicitly configured. Will be separated when IT tests become too slow.

## Consequences
- H2 dependency removed from `pom.xml`
- `src/test/resources/application.yml` emptied — `@ServiceConnection` overrides
  datasource automatically
- `TrainingMetricsApiApplicationTests` migrated to Testcontainers
- `*IT.java` test classes cover: generated IDs, FK constraints, derived queries,
  Flyway migrations (implicit — container starts from V1)
- E2E tests (`*IT.java` with `WebEnvironment.RANDOM_PORT`) planned after
  Spring Security implementation (TDD-005)

## Dependencies added
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-testcontainers</artifactId>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.testcontainers</groupId>
    <artifactId>testcontainers-postgresql</artifactId>
    <scope>test</scope>
</dependency>
```
Note: versions managed by Spring Boot BOM. `testcontainers-postgresql` is the
correct artifactId for Testcontainers 2.x (breaking change from 1.x `postgresql`).
